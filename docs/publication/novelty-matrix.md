# Novelty matrix

_Last updated: 2026-07-17_

This matrix separates established patterns from claims that still require evidence. It is not manuscript prose.

| Dimension | Established prior pattern | Current repository artifact | Potential distinct contribution | Evidence still required |
|---|---|---|---|---|
| Execution-trace visualization | Trace visualizations have been evaluated against ordinary IDE use using correctness and time | Source-pinned authored GGUF trace and keyboard-operable viewer | Evidence-labelled coordination of source, call stack, runtime objects, tensor shapes, memory events, and explanations for an evolving AI-systems codebase | Information-equivalent static baseline, frozen benchmark, learner/expert evaluation |
| Executable lectures | CS336 separates trace-producing lecture programs from a frontend viewer | Deterministic trace schema, replay validator, static transcript, narrow frontend | Revision-pinned educational trace lifecycle with explicit captured/source-derived/authored boundaries | Real or faithfully captured trace, independent correctness review |

| Reproducible learning environments | Browser notebooks, lockfiles, containers, and hosted workspaces are established delivery mechanisms | Deliberately unequal browser, local-native, and devcontainer tiers | Making runtime/evidence differences a learning objective rather than hiding them | Executed reproducibility matrix and timing records |
| Repository-native AI-agent workflow | Agent-assisted documentation and software maintenance are active areas | Structured assignments, run logs, validators, corrections, CI and human gates | Longitudinal case study of scheduled specialist agents maintaining source-linked education for a moving systems codebase | DATA-01 extraction, simpler-workflow baseline, human labor/cost accounting |

## Claim boundary

The repository currently demonstrates an architecture and validated artifacts. It does not yet demonstrate educational benefit, workflow superiority, or generalization beyond llama.cpp.

The hypothesized gap between usage documentation and source-level systems understanding is not a novelty claim until a systematic documentation audit supports it.
