# Executable-learning two-week execution plan

_Last updated: 2026-07-17 17:58 Africa/Cairo_

This dependency-ordered plan covers July 17-31, 2026. It is narrower than the long-term interactive-textbook roadmap and does not authorize manuscript drafting or participant recruitment.

## Frozen target and learning progression

Primary learners are advanced undergraduates and beginning graduate students who can read C/C++ and Python but cannot yet connect GGUF storage, virtual memory, GGML graph construction, backend scheduling and token generation.

The three experiences form one progression:

1. **Lab 0 — Build and Run llama.cpp:** identify and diagnose environment, configure, compile, executable-launch, model-load and inference stages.
2. **Lab 1 — GGUF Anatomy:** predict, parse and explain bounded GGUF layout while distinguishing browser-derived file evidence from native runtime behavior.
3. **Executable Lecture 0 — GGUF loading trace:** reconstruct a pinned source path while distinguishing authored, source-derived and captured evidence.

Every experience declares intended learner, prerequisite, learning objective, predicted misconception, executable action, observable output, formative assessment, source revision, validation and accessibility fallback.

## Platform tiers

| Tier | Purpose | Required capability | Explicit non-claim |
|---|---|---|---|
| Browser/static | zero-install concepts, fixtures, checkpoints and local progress | deterministic parsing/visualization, keyboard/static fallback | native llama.cpp execution |
| Local native | real toolchain and bounded native experiments | `uv sync --locked`, CMake/Ninja, compiler checks, model-free smoke test | uniform performance or model inference without evidence |
| Cloud container | reproducible one-click workspace | devcontainer/Codespaces-compatible toolchain and bounded smoke test | free unlimited compute or permanent storage |

No restricted model weights are redistributed. The core path is model-free; optional inference uses a clearly documented learner-provided model path.

## Authoritative evidence rules

- Deterministic SVG/Mermaid/Graphviz/D3 or trace-derived figures are authoritative.
- Optional generated images/audio/video are supplemental, manifest-driven, cached, secret-safe, manually triggered or review-gated, accessibility-complete and human-approved.
- Ordinary pushes never spend API budget or silently regenerate approved media.
- Browser-derived, authored, source-derived and native-captured evidence remain visibly distinct.
- Local progress is anonymous, versioned and exportable; no authenticated sync or silent telemetry is approved.
