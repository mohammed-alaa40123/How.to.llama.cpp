# EAAI publication roadmap

_Last updated: 2026-07-17 17:58 Africa/Cairo_

This roadmap turns repository milestones into evidence gates for an EAAI experience report. It does not authorize manuscript drafting.

## Phase A — Foundation, July 17-23, 2026

### A1. Coordination and contracts

- Freeze the primary audience, educational problem, three lesson contracts and research questions.
- Maintain authoritative state, dependency queue, handoffs and readiness scorecard.
- Keep browser, local-native and cloud-container tiers intentionally unequal and explicitly labelled.
- Preserve Verified, Interpretation, Historical and Open Question claim classes.

**Status:** substantially evidenced. The remaining foundation dependency is STACK-01: consolidate the stacked PR chain into one human-reviewable integration path without losing failed or superseded work.

### A2. Deterministic evidence foundations

- Legal fixture boundary and deterministic synthetic GGUF.
- Lab 0 report/reproducibility contract.
- Trace schema, source anchors, sample authored trace and deterministic replay.
- Local progress schema and portability implementation.
- Media manifest/provenance and lifecycle contracts.
- Strict documentation and accessibility-structure CI.

**Status:** evidenced as bounded components. This does not establish clean-environment reproducibility, independent correctness or educational effectiveness.

### A3. Smallest vertical slice

- Browser GGUF parser/visualizer with Predict-Discover-Explain checkpoints.
- Keyboard-operable authored/source-derived trace viewer.
- Deterministic technical figure.
- Local progress resume/export/import integration.
- Information-equivalent benchmark contract.

**Status:** evidenced through component CI. The artifacts are not yet one canonical end-to-end branch.

## Phase B — Vertical slices, July 24-31, 2026

### B1. Canonical integration and Lab 0

1. Complete STACK-01 merge map and choose the canonical progress implementation.
2. Execute LAB0-03 on clean Ubuntu 24.04 using `uv sync --locked`, CMake/Ninja and the bounded model-free target.
3. Execute LAB0-04 in the devcontainer/Codespaces-compatible environment using the same evidence schema.
4. Preserve exact failures, degraded/offline states and time-to-ready; never label executable launch as model loading or inference.

**Exit evidence:** one canonical integrated branch, passing CI and measured local/container records with exact revisions and commands.

### B2. Coherent learning progression

- Define DEMO-01A acceptance criteria that connect Lab 0, Lab 1 and the executable lecture through one learner journey.
- Require each experience to expose prerequisite, objective, misconception, executable action, observable output, formative assessment, source revision, validation and accessibility fallback.
- Demonstrate save/export/clear/import/resume without treating reached steps as mastery.
- Retain a complete static/text route through the browser lab and trace viewer.

**Exit evidence:** locally reproducible or deployed demo with known limitations, not merely a list of component links.

### B3. Evidence package preparation

- Complete independent technical review of fixture, browser parser/explanations, source anchors, figure and benchmark answer key.
- Freeze DATA-01B coding/missing-value rules and extract a broader retrospective sample.
- Choose an evaluation pathway; do not recruit or collect personal data before approval and any required ethics review.
- Keep optional generated media outside the critical path.

**Exit evidence:** corrections retained, case-study data auditable and evaluation pathway approved.

## Phase C — Evidence development, August-September 2026

- Run the approved information-equivalent baseline comparison.
- Collect learner or expert evidence only under the approved pathway.
- Analyze setup completion, time-to-ready, bounded code-tracing correctness, confidence calibration, cognitive load and accessibility completion mode.
- Analyze agent assignments, blockers, duplicate work, CI repairs, human corrections, accepted/revised/rejected outputs, labor, cost proxies and infrastructure burden.
- Report negative and null findings.

**Exit evidence:** measured educational/reproducibility outcomes, broader longitudinal data and independent correctness review.

## Generalization boundary

The artifact is llama.cpp/GGML-specific. The potentially generalizable method is source-pinned executable explanation, evidence-labelled traces, deterministic validators, local-first learner state and human-supervised repository agents. Generalization beyond this codebase remains a design hypothesis until replicated or compared elsewhere.

## Ethics and AI-use boundary

- No participant recruitment or personal-data collection without explicit approval and required review.
- No silent telemetry or authenticated synchronization in the two-week demo.
- Deterministic technical graphics are authoritative.
- Generated images, audio or video are optional supplements with provenance, accessibility, licensing/privacy review, caching and human technical approval.
- Manuscript AI-use disclosure must report agent roles, tools/models when known, human corrections, validation, failures, duplicate work, costs and retained evidence.
